# Serendipity Experience Detection in Amazon Online Reviews with Support Vector Machnie and Convolutional Neural Network

## Project Overview
Investigating **serendipitous recommender systems** is becoming a common trend in **Artificial Intelligence (AI)** research to provide serendipitous recommendations for consumers. 
However, most studies in this field have not addressed the creation of datasets using online user reviews on e-commerce platforms, as labelling serendipity experiences in online reviews requires substantial time and financial costs.
This independent project aims to critically evaluate an **SVM with TF-IDF** and a **CNN with Word2Vec** for detecting serendipity experiences in online user reviews.

## Data
### Raw data
- **SerenLens** dataset (`SerenLens_Books.csv`), created by Fu et al. (SIGIR 2023).
- Source: https://github.com/zhefu2/SerenLens
- Columns:
    - `user_id` : the id number of users.
    - `item_id` : the id number of items.
    - `timestamp` : the timestamp of the review.
    - `review` : the content of the reviews.
    - `rating` : the 5-star rating of the reviews.
    - `label`: the label of serendipitous feelings for the reviews, 0 denotes not serendipitous feeling, 1 denotes serendipitous feeling.
### Pre-processed data
- `1_preprocessing.ipynb` produced `SerenLens_preprocessed.csv` for training and validating the SVM and CNN, stored in `data/processed/`.
- Columns:
    - `clean_text` : the review text after preprocessing (lower-casing, tokenisation, stop-word removal, etc.)
    - `label` : the label of serendipitous feelings for the reviews, 0 denotes not serendipitous feeling, 1 denotes serendipitous feeling.
    - (The other raw columns - `user_id`, `item_id`, `timestamp`, `rating` - were dropped, allowing for **privacy and ethical concerns**.)
